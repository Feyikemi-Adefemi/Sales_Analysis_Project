# 📊 Sales Analysis Project

## Project Overview

This Sales Analysis Project is one of the biggest things I have worked on so far in my data analytics journey. I built it step by step using Microsoft Excel, Python, SQL, and Power BI. The whole idea was not just to make charts or write code, but to really understand how a real business dataset works and how to turn raw data into something useful.

I used a retail sales dataset and looked at things like total sales, customer behavior, product performance, regional performance, and sales trends over time. What made this project special for me was that I did the same analysis in different tools, so I could see how Excel, Python, SQL, and Power BI all connect to each other. That helped me understand the data better and also helped me build confidence.

This project was part of my personal journey to become a Data Analyst. I wanted to learn the process properly, not just rush to the final answer. So I took my time, made mistakes, fixed them, and learned from each stage.

---

# Project Objectives

The main things I wanted to achieve with this project were:

* Understand the structure of a real dataset and see what kind of information it contains.
* Clean the data and check for problems before starting the analysis.
* Calculate important business numbers like total sales, number of orders, and average sales.
* Look at sales performance by region, category, customer, product, and year.
* Build dashboards that make the data easy to understand.
* Compare how the same analysis looks in Excel, Python, SQL, and Power BI.
* Write down what I learned at each stage in a clear way.
* Create a full project that I can add to my portfolio and show to others.

# Excel Analysis

Excel was the first place I started the analysis. This was where I got familiar with the dataset and checked if the data was in good shape before moving to other tools.

In Excel, I focused on understanding the data, cleaning it, and calculating the main KPIs. I also used Pivot Tables to summarize the data and make it easier to see patterns. This part of the project helped me build a strong base because I could see the numbers clearly before moving into Python or SQL.

Key tasks included:

* Data validation
* Duplicate checks
* Missing value identification
* KPI calculations
* Pivot Tables
* Dashboard creation

This stage taught me how important it is to slow down and check the data properly. Even small issues in Excel can affect the rest of the analysis if they are not handled early.

---

# Python Analysis

Python was where I started to explore the data more deeply. I used it for cleaning, calculating KPIs, and creating visualizations. This part of the project helped me move beyond spreadsheets and start thinking more like an analyst.

With Python, I could see the data in a different way. I imported the dataset, checked the structure, cleaned it, and then used Matplotlib to create charts. I also used Python to compare results with Excel so I could make sure the numbers matched.

Activities included:

* Data import
* Data cleaning
* KPI calculations
* Data visualization using Matplotlib
* Business insights
* Documentation

Python felt a little challenging at first, but once I started understanding the logic, it became easier. I liked that I could write code and get results quickly, especially when I wanted to test ideas or check patterns in the data.

---

# SQL Analysis

SQL was one of the most useful parts of the project because it showed me how to work with data in a database. I used SQLite to query the imported dataset and repeat the same analysis I had already done in Excel and Python.

This helped me understand that SQL is not just about writing queries. It is really about asking the right questions from the data and getting answers in a clean and structured way.

Key SQL concepts included:

* SELECT
* FROM
* SUM()
* COUNT()
* DISTINCT
* GROUP BY
* ORDER BY
* LIMIT

One thing I learned during this stage was that SQL can be very powerful, but it also depends on how the data is stored. I had to pay attention to the format of the dates and how the table was imported. That taught me to always check the data first before assuming the query is wrong.

# Power BI Dashboard

Power BI was the part where everything started to come together visually. After doing the analysis in Excel, Python, and SQL, I used Power BI to build an interactive dashboard that could tell the story of the data in a simple way.

I wanted the dashboard to be clear, useful, and easy to explore. So I added KPI cards, charts, and slicers that allow someone to look at the data by region, category, year, customer, and product.

Dashboard features included:

* KPI Cards
* Sales by Region
* Sales by Category
* Sales Trend by Year
* Top Customers
* Top Products
* Interactive slicers

This part of the project was exciting because it made the analysis feel more real. Instead of just looking at tables and code, I could see the business story in one place. It also helped me understand how dashboards are used in real business settings.

# Key Business Insights

After going through all the analysis, I found a few important things:

* The West region generated the highest sales revenue.
* Technology was the best-performing product category.
* Sales kept growing over the years, and 2018 had the highest annual sales.
* A small group of customers brought in a large share of the total revenue.
* Some products performed much better than others and contributed a lot to the business.

These findings showed me that the business had strong areas, but also some places where growth could still happen. It was interesting to see how the same dataset could tell a clear story once it was properly analyzed.

---

# Recommendations

Based on what I found, these are the things I would suggest:

* Use the strategies that worked well in high-performing regions and try them in weaker regions.
* Keep investing in Technology products since they performed the best.
* Build stronger relationships with high-value customers.
* Keep tracking sales trends so the business can plan better.
* Use dashboards regularly so decision-making becomes easier and faster.

These recommendations came directly from the data, not from guessing. That made the project feel more useful and realistic.

# Challenges Encountered

Like every real project, this one came with a few problems along the way. Some of them were small, but they still mattered because they affected the final result.

Some of the challenges I faced were:

* Handling missing postal code values.
* Correcting data type mismatches during Power BI import.
* Resolving DAX calculation errors.
* Checking KPI differences across different tools.
* Fixing SQL date-format issues caused by imported text values.

What I learned from these challenges is that data work is not always smooth. Sometimes the issue is not the tool itself, but the way the data is stored or imported. I had to stop, check carefully, and solve each problem one by one before moving on.

That process taught me patience and made me more confident. Instead of getting stuck, I learned how to troubleshoot and keep going.
